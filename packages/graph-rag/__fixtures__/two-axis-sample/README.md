# Two-Axis Sample — Graph RAG mock data

Frozen sample of the two knowledge artifacts the retrofitted reader consumes
(per Two-Axis ADR D7). Used to design and test the reader against realistic data
without touching the live repo.

## Layout mirrors the real repo, under a fixture root

```
__fixtures__/two-axis-sample/          ← the "root" you point the reader at
├── .agents/changelogs/archive/
│   ├── index.json                     ← time axis (stories + epics)
│   └── schema.json                    ← archive schema (for validation)
├── apps/srs-demo/KNOWLEDGE.md         ← domain axis (routing, app-shell)
└── packages/srs-engine-v2/KNOWLEDGE.md ← domain axis (batch-composition)
```

The reader must take the root as a parameter — production points it at the repo
root, tests point it here. Paths here are **not** the live paths; the only
contract is that `index.json` sits at `.agents/changelogs/archive/` and each
unit's `KNOWLEDGE.md` sits at its workspace-relative path (`apps/*`, `packages/*`)
below the root. A `KNOWLEDGE.md`'s own unit is its `unit:` frontmatter, not its
disk location.

## Provenance

Generated by a sample run of the `archive-epic` skill over **EP44** (App.vue →
Vue Router refactor, PR #41), stopped before the compact/delete step. It is a
point-in-time snapshot — do not re-sync from the live artifacts; regenerate
deliberately if the schema changes.

## What it exercises

- Two `domain` nodes from real workspace units — grouping is by workspace, never
  by epic.
- `epic` (EP44) appears **only as an edge target** of `sources` provenance.
- Provenance at both grains: `apps/srs-demo → EP44` (epic-level) and
  `packages/srs-engine-v2 → EP44-RV01` (story-level).
- A `fixes` edge (`EP44-RV01 → EP23`) and a cross-track entry (`AGN05`, agentic).
