## Summary

Implemented `processAnswers` on `SrsEngine` ‚Äî orchestrates mastery update, FSRS scheduling, foundational wrong rule, stuck-word detection, and shelving. Added 32 tests (config validation unit tests + 5 integration lifecycle scenarios).

## Files Modified

### `packages/srs-engine/src/srs-engine.ts`
- `processAnswers(answers, wordStates)` ‚Äî per answered word:
  1. `updateMastery` ‚Äî mastery count + phase transition
  2. FSRS `scheduleReview` ‚Äî only when word was `srsM2_review` **before** the answer (avoids scheduling freshly promoted words)
  3. `applyFoundationalWrongRule` ‚Äî only when word was `foundational` + `learning` **before** the answer (avoids double-penalising srsM2_review demotions)
  4. Reset `consecutiveWrongCount ‚Üí 0` on foundational correct
  5. `batchesSinceLastProgress` tracking ‚Äî learning-phase words only; reset to 0 on mastery progress, increment otherwise
  6. `detectStuckWords` on full updated array ‚Üí shelve eligible words (24h duration constant)

### `packages/srs-engine/src/__tests__/srs-engine.test.ts` (new)

- **composeBatch**: 3 unit tests (basic batch, shelved exclusion, newWordsPerBatch limit)
- **Scenario 1** ‚Äî Learning ‚Üí srsM2_review: promotion at threshold, no FSRS on fresh promotion
