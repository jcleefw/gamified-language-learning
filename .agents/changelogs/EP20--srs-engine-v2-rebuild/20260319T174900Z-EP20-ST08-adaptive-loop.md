# EP20-ST08: Adaptive loop — sliding window, mastery, retirement, run-until-exhausted

**Created**: 20260319T174900Z
**Epic**: [EP20 - SRS Engine v2: Rebuild from Scratch](../../plans/epics/EP20-srs-engine-v2-rebuild.md)
**Status**: Complete ✅

## Summary

Replaced the pre-computed `generateBatches` + `runBatchLoop` architecture with a single
`runAdaptiveLoop` that owns the full learning lifecycle. Words move through a
queue → active → mastered cycle. The run continues until all words are mastered and
both active pool and queue are empty.

Implements the sliding active window described in the Learning Phase ADR.

## Files Added

### `packages/srs-engine-v2/src/__tests__/unit/adaptive-loop.test.ts`

- 6 tests for `nextActivePool`:
  - Returns unchanged active + queue when no words are mastered and active is full
  - Retires a mastered word and pulls next from queue
  - Retires multiple mastered words and pulls same number from queue
  - Returns empty active and queue when queue is exhausted after retirement
  - Returns active shorter than questionLimit when queue runs out
  - Does not mutate input arrays

## Files Modified

### `packages/srs-engine-v2/src/types/word-state.ts`

- Added `isMastered(ws: WordState, threshold: number): boolean` — `ws.correct >= threshold`

### `packages/srs-engine-v2/src/__tests__/unit/word-state.test.ts`

- Added 4 tests for `isMastered`:
  - Returns false when `correct < threshold`
  - Returns true when `correct === threshold`
  - Returns true when `correct > threshold`
  - Returns false on a fresh `WordState`

### `packages/srs-engine-v2/src/runner/interactive.ts`

- Removed `runBatchLoop`
- Added `nextActivePool(active, queue, questionLimit, runState, masteryThreshold)` — exported pure function
  - Retires mastered words from active
  - Fills free slots from front of queue
  - Returns new `{ active, queue }` without mutating inputs
- Added `runAdaptiveLoop(words, wordPool, foundationalPool, questionLimit, masteryThreshold)`
  - Initialises active=[], queue=[...words], runState=Map
  - Each iteration: fills active via `nextActivePool`, composes questions, runs quiz,
    updates RunState, prints per-word summary, announces newly mastered words
  - Splits active by type (`'class' in item`) and routes each to the correct full pool
    for distractor generation (fixes 2-choice regression from using active as pool)
  - Peeks next state to decide whether to prompt "Next batch? (y/n)"
  - Terminates automatically when active and queue both empty
  - Prints run summary: batches, score, mastered count

### `packages/srs-engine-v2/src/main.ts`

- Replaced `Deck` + `BatchConfig` + `generateBatches` + `runBatchLoop` with flat config
- Config includes both `mockWords` and `mockConsonants` in the queue (mastery applies to both)
- Passes `mockWords` and `mockConsonants` as full pools for distractor generation

### `packages/srs-engine-v2/src/types/deck.ts`

- Removed `BatchConfig` and `Batch` interfaces (superseded by adaptive loop)
- `Deck` interface retained

## Files Removed

## Behavior

- All 39 tests pass (2 smoke + 26 compose-batch + 4 isMastered + 6 nextActivePool + 1 updateRunState)
- `pnpm quizv2` — run starts with active pool filling from queue up to questionLimit;
  new words enter when a slot opens (on mastery); "Mastered: X" printed on graduation;
  run ends automatically when all words mastered with a final summary
- Both foundational (consonants) and non-foundational (words) go through the same mastery cycle
- All MC questions have 4 choices (distractor pool is full word/consonant pool, not just active)

## Regression Fixes (post-implementation review)

Two regressions from initial ST08 implementation were caught in review and fixed:

1. **Foundational words removed** — `main.ts` initially only queued `mockWords`. Fixed by
   including `mockConsonants` in the queue and passing both full pools to `runAdaptiveLoop`.

2. **Only 2 MC choices** — `composeBatchMulti(active, active, ...)` used the active pool
   (2 items) as the distractor source. Fixed by splitting active by type and passing the
   full `wordPool` / `foundationalPool` for distractor generation.
