## 1. Feature Overview

srs-demo currently loads all deck/word/sentence content from a hardcoded TypeScript file (`src/data/decks.ts`). This spec replaces that with DB-backed content across four stories:

1. **`importCurriculum`** ‚Äî a new util in `@gll/db` that inserts `AppDeck[]` content into the DB tables, generating UUIDs for decks/sentences and using INSERT-OR-IGNORE + read-back for words (ADR-canonical dedup). Fixes three bugs present in the cli-demo-db version (see table below).
2. **Server seeder** ‚Äî on `GLL_SEED_CONTENT=1`, reads `conversations-2026-03-08.json` (5 topics), transforms each topic to `AppDeck`, and calls `importCurriculum`. No custom seed file ‚Äî the JSON fixture is the seed source. Gives 5 selectable decks during manual testing.
3. **API routes** ‚Äî `GET /api/decks` (serves content to the Vue app); `POST /api/curriculum/import` (curator upload endpoint, accepts **one** `ConversationJSON` per request).
4. **Vue app rewrite** ‚Äî fetches decks from the API on mount; uses UUID deck IDs throughout (including shelving calls); deletes `decks.ts`, `transformer.ts`, `types.ts`.

**Fixes applied to the cli-demo-db import pattern:**

| Issue | cli-demo-db behaviour | Correct behaviour in `@gll/db` |
|---|---|---|
| Deck ID | `conv.id` ‚Üí `undefined` (JSON has no `id` field) ‚Üí stored as `NULL` | Always `randomUUID()` at import time |
| Speaker | Hardcoded `null` ‚Äî `conv.lines[i].speaker` never read | Read from `AppLine.speaker` |
| Word dedup | UUID generated in memory, trusted as stored | `INSERT OR IGNORE` on `(language, text)`, then read-back UUID for FK use |

**ID strategy** (per DB schema ADR):

| Entity | ID | Source |
|---|---|---|
| `decks.id` | UUID | Generated by `importCurriculum` |
| `words.id` | UUID | Generated on first insert; read back for subsequent FK use |
| `sentences.id` | UUID | Generated by `importCurriculum` per line |
| `sentence_components.id` | UUID | Generated by `importCurriculum` per component |
| `users.id` | `'demo-user'` sentinel | Seeded on startup |

Deck IDs are UUIDs ‚Äî the `'deck-eat'` / `'deck-weather'` strings used in DS01‚ÄìDS02 are replaced. The Vue app reads UUID deck IDs from `GET /api/decks` and threads them through to shelving API calls.

## 2. Core Requirements

| Requirement | Decision | Rationale |
|---|---|---|
| `importCurriculum` location | `packages/db/src/import-curriculum.ts` | Pure DB operation; reusable by server and any future CLI tool |
| `AppDeck`, `AppLine`, `AppWord` and payload types | `@gll/api-contract/src/content.ts` | HTTP wire types for both API routes and Vue app consumption; `@gll/db` imports from there |
| `ConversationJSON` types | `@gll/api-contract/src/content.ts` | Canonical definition of the raw curator upload format |
| `transformConversation` function | `apps/server/src/transform-conversation.ts` | Single-topic transform; keeps `@gll/api-contract` types-only |
| `POST /api/curriculum/import` body | Single `ConversationJSON` | One topic per upload ‚Äî production curator pattern; seeder loops over the JSON array internally |
| Seed source | `packages/srs-engine-v2/data/samples/conversations-2026-03-08.json` | 5 topics ‚Üí 5 decks; exercises the same transform + import path as production |
| No custom seed file | `srs-demo-content.ts` not created | JSON fixture is the seed source; no transcription of `decks.ts` |
| Word dedup | INSERT OR IGNORE on `(language, text)`, read back UUID | ADR-canonical; safe across multiple imports of overlapping vocabulary |
| Seed trigger | `GLL_SEED_CONTENT=1` env var | Opt-in ‚Äî not on every cold start; `seedDemoUser` always runs as it is idempotent |
| Deck IDs in Vue app | UUID from `GET /api/decks` response | Stored in `deckId` ref; passed to shelving API calls; persisted in `localStorage` as UUID |
| `decks.ts` / `transformer.ts` / `types.ts` | Deleted from `apps/srs-demo/src/data/` | Replaced by API fetch; types move to `@gll/api-contract` |

**`importCurriculum` signature** (`packages/db/src/import-curriculum.ts`):

```typescript
export function importCurriculum(db: DbClient, decks: AppDeck[]): void
// Per deck: randomUUID() for decks.id
// Per word (deduped by native + language across all lines):
//   INSERT OR IGNORE INTO words; SELECT id FROM words WHERE language=? AND text=? ‚Üí read-back UUID
// Per line: randomUUID() for sentences.id; speaker from AppLine.speaker; onConflictDoNothing on (deck_id, text)
// Per word-in-line: randomUUID() for component id; word_id = read-back UUID; checks sentence_components before inserting
// Per deck: populate deck_words from collected word UUIDs
```

**`transformConversation` signature** (`apps/server/src/transform-conversation.ts`):

