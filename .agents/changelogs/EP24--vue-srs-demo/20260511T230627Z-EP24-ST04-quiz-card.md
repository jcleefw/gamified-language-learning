## Implementation

Props: `question: QuizQuestion`, `index: number`, `total: number`.
Emits: `answered(result: QuizResult)`.

Local state:
- `answered: ref(false)` — gates repeated clicks
- `selectedLabel: ref<string | null>(null)` — tracks which button was clicked for CSS feedback

On choice click: sets `answered = true`, stores `selectedLabel`, emits `{ wordId, correct: choice.isCorrect }`.

CSS classes applied after answer:
- `.correct` on the correct choice (green)
- `.wrong` on the selected incorrect choice (red)
- `.disabled` on all buttons (cursor: default)

A `watch` on `props.question` resets `answered` and `selectedLabel` when the parent advances to the next question — necessary because Vue reuses the component instance rather than recreating it.

## Decisions

- Local `answered` flag rather than relying on the parent to immediately remove the component — gives the user a moment of visual feedback before the transition.
- Direction label rendered as human-readable text via `.replace(/-/g, ' → ')` (e.g. `native → english`).
