# EP26-ST05: Host Integration Wiring

**Created**: 20260626T130504Z
**Epic**: [EP26 - SRS Shelving Policy](../../plans/epics/EP26-srs-shelving-policy.md)
**Status**: Complete ✅

## Summary

Wired the DS02 shelving pipeline into all three hosts: Hono server (API routes), srs-demo (Vue), and CLI demo. Replaced DS01 in-memory stagnation detection (`recordMasterySnapshot`/`detectStagnantWords`) with the DB-layer counter approach throughout. All API routes and composables are now deck-scoped.

## Files Modified

### `packages/api-contract/src/srs.ts`
- Added `deckId` to `ApplyShelvingRequest`
- Added `UnshelveAllRequest`, `UpdateStagnationCountersRequest`, `GetStagnantWordsResponse`, `ResetStagnationCountersRequest` types

### `apps/server/src/routes/shelving.ts`
- `GET /api/shelving?deckId=` — deck-scoped

- **New**: `POST /api/stagnation/update`
- **New**: `GET /api/stagnation/stagnant?deckId=&threshold=`
- **New**: `POST /api/stagnation/reset`

### `apps/srs-demo/src/composables/useShelving.ts`
