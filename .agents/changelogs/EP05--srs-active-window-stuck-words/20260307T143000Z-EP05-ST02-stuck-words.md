# EP05-ST02: Stuck Word Detection + Shelving

**Created**: 20260307T143000Z

## Summary

Implemented stuck word detection and shelving system for the SRS engine. Words that show no mastery progress for 3 consecutive batches are marked as "stuck" and shelved for 1 day to allow cognitive reset. System respects a max-2 shelved cap, with overflow logic (newest stuck word is shelved when capacity is reached). All functions are immutable and time-aware.

## Files Created

### `packages/srs-engine/src/stuck-words.ts`
- `detectStuckWords(wordStates, config): StuckWordsResult` — Identifies words with no mastery progress for N consecutive batches (N = `config.shelveAfterBatches`, default 3). Returns stuck words, prioritized words to shelve (respecting max-2 cap), and availability flag.
- `shelveWord(word, durationMs): WordState` — Sets `shelvedUntil` timestamp (default: 86400000ms = 1 day). Returns immutable new state.
- `unshelveWord(word): WordState` — Clears `shelvedUntil`. Returns immutable new state.
- `isShelved(word): boolean` — Time-aware check: returns true if word is currently shelved (future date, not expired).

## Behavior Preserved / New Behavior

**New Behavior**:
- Words stuck for 3+ batches without mastery progress are identified
- Stuck words are shelved for 1 day (86400000ms)
- Max 2 words can be shelved simultaneously; when cap is reached, only newest stuck word is shelved
- Shelved words become available for re-entry after shelve duration elapses
- `isShelved()` correctly evaluates both current shelve status and time-based expiry

**Type Extensions** (in `src/types.ts`, already in place from EP05-ST01):
- `WordState.batchesSinceLastProgress?: number` — Tracks consecutive batches without mastery progress
- `WordState.shelvedUntil?: Date | null` — Shelve expiry timestamp (null = not shelved)
