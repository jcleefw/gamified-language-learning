## 1. Feature Overview

Two interconnected modules enforce learning efficiency constraints in the SRS engine:

1. **Active Window Management** (`active-window.ts`): Limits concurrent active words to 8, caps new word introduction at 4 per batch, and enables sliding entry as words advance to review phase (srsM2_review)
2. **Stuck Word Detection & Shelving** (`stuck-words.ts`): Detects words with no mastery progress over 3 consecutive batches, shelves them for 1 day to prevent indefinite rotation, with max 2 shelved simultaneously and re-entry as carry-over

Together these prevent cognitive overload and unblock learners stuck on difficult words.

---

## 2. Core Requirements

| Requirement | Decision | Rationale |
|-------------|----------|-----------|
| Active ceiling | 8 words max | Balance depth (mastery) vs breadth (vocabulary); prevents cognitive overload |
| New words per batch | Max 4 per batch | Prevents overwhelming learner with too many simultaneous introductions |
| Window entry trigger | Advance to srsM2_review phase | Word has demonstrated mastery; ready for SRS scheduling |
| Stuck detection threshold | 3 consecutive batches, no mastery progress | Sufficient attempts to identify persistent gaps; early enough to intervene |
| Shelve duration | 1 day (86400000 ms) | Allows cognitive reset without permanent loss |
| Shelved capacity | Max 2 words | Prevents explosion of recovery state; bounds complexity |
| Shelved overflow behavior | Newest stuck word shelved; 3rd queued | Simple FIFO; prevents livelock; doesn't displace existing work |
| Re-entry mechanism | Carry-over on next eligible batch | Gradual reintroduction; counts toward 4-new-per-batch cap |
| Progress reset | On mastery count increase | `batchesSinceLastProgress` resets to 0 when word advances |

## 3. Data Structures

### WordState Extensions

`WordState` type (in `src/types.ts`) requires two new optional fields:

```typescript
export interface WordState {
  wordId: string
  category: WordCategory
  masteryCount: number
  phase: MasteryPhase
  lapseCount: number
  correctCount: number
  wrongCount: number
  fsrsState?: FsrsCardState

  // NEW — EP05:
  batchesSinceLastProgress?: number  // Batches with no mastery progress; resets on progress
  shelvedUntil?: Date | null         // null = not shelved; Date = unshelve at this UTC time
}
```

### Active Window Types

```typescript
// packages/srs-engine/src/active-window.ts

export interface EligibleWordsResult {
  active: WordState[]        // Words currently in active window (phase tracking)
  newSlots: number           // Slots available for new words (0–4)
  eligible: WordState[]      // Candidate new words to fill slots
}

export function getEligibleWords(
  allWords: WordState[],
  config: SrsConfig,
): EligibleWordsResult
```

### Stuck Words Types

```typescript
// packages/srs-engine/src/stuck-words.ts

export interface StuckWordsResult {
  stuck: WordState[]         // Words identified as stuck (3+ batches, no progress)
  toShelve: WordState[]      // Words to be shelved (respects max-2 cap)
  canReShelve: boolean       // true if shelved count < 2; can accept more
}

export function detectStuckWords(
  wordStates: WordState[],
  config: SrsConfig,
): StuckWordsResult

export function shelveWord(
  word: WordState,
  durationMs: number,
): WordState

export function unshelveWord(word: WordState): WordState

export function isShelved(word: WordState): boolean
```

## 4. User Workflows

### Active Window Entry Flow

```
Learner Completes Batch
  ↓
Check each word: mastery progress?
  ├─ YES (masteryCount increased)
  │   └─ Word advances to srsM2_review phase
  │       └─ Next batch: check active window
  │           ├─ < 8 active words?
  │           │   └─ Enter active window
  │           └─ ≥ 8 active words?
  │               └─ Queue for next available slot
  │
  └─ NO
      └─ Remains in learning or review phase
```

### Stuck Word Detection & Shelving Flow

```
Batch Completed
  ↓
Update word_states (increment counters, track progress)
  ↓
For each word: mastery progress since last batch?
  ├─ YES
  │   └─ Reset batchesSinceLastProgress = 0
  │
  └─ NO
      └─ Increment batchesSinceLastProgress
          └─ >= 3?
              └─ Mark as stuck
                  └─ Current shelved count < 2?
                      ├─ YES → Shelve immediately (shelvedUntil = now + 1 day)
                      └─ NO → Queue; shelve when slot frees
```

### Shelved Word Re-entry Flow

```
Current time >= word.shelvedUntil?
  ├─ YES
  │   └─ Unshelve word (clear shelvedUntil)
  │       └─ Add to carry-over on next batch
  │           └─ Counts toward 4-new-per-batch cap
  │
  └─ NO
      └─ Remains shelved
```

## Notes

- **Newest Stuck Word**: When shelve cap is reached, the most-recently-detected stuck word is shelved (design decision confirmed 2026-03-05)
- **Carry-over Semantics**: Re-entered words count toward the 4-new-per-batch cap, not as "extra" additions
- **Date vs Timestamp**: Use `Date` objects for `shelvedUntil`; convert to UTC in config if needed
- **Batch Counter Reset**: `batchesSinceLastProgress` should reset whenever `masteryCount` increases (handled by calling layer after update)
- **No Wiring**: Active window and stuck words are utilities; batch composition (EP04/EP07) integrates them
